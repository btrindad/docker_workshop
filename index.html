Lab 07

the docker file

build your image with a docker file

Dockerfile


FROM ubuntu:14.10

RUN \
apt-get update && \
apt-get install -y nginx && \
rm -rf /var/lib/apt/lists/* && \
echo "\ndaemon off;" >> /etc/nginx/nginx.conf && \
chown -R www-data:www-data /var/lib/nginx

CMD ["nginx"]

EXPOSE 80
            

# create a Dockerfile
docker build -t yourname/nginx:v3 the/location/of/the/docker/file
docker run -p 80:80 -d yourname/nginx
# go checkout your web site http://localhost

what did we just do?

  • build the new image from ubuntu 14.10 
    
FROM ubuntu:14.10
  • run these commands on the new container 
    
RUN \
apt-get update && \
apt-get install -y nginx && \
rm -rf /var/lib/apt/lists/* && \
echo "\ndaemon off;" >> /etc/nginx/nginx.conf && \
chown -R www-data:www-data /var/lib/nginx
  • run the nginx comand when you start the file 
    
CMD ["nginx"]
  • expose port 80 to the host(the host must still bind this to a port) 
    
EXPOSE 80
  • comand to build the image from the docker file 
    
docker build -t yourname/nginx:v3 the/location/of/the/docker/file

now we can build a image on the fly but how do we get our data in there

Lab 09

using volumes

use a named volume


FROM ubuntu:14.10

RUN \
apt-get update && \
apt-get install -y nginx && \
rm -rf /var/lib/apt/lists/* && \
echo "\ndaemon off;" >> /etc/nginx/nginx.conf && \
chown -R www-data:www-data /var/lib/nginx

COPY Sitefile /etc/nginx/sites-enabled/

CMD ["nginx"]

EXPOSE 80
            

# create a Dockerfile
docker build -t yourname/nginx:v5 the/location/of/the/docker/file
docker run -d -p 80:80 -v $(pwd):/var/www yourname/handbuiltnginx:v5 service nginx start
# go checkout your web site http://localhost

what did we just do?

  • link the local directory in the container 
    
-v $(pwd):/var/www yourname/handbuiltnginx:v5

but this relies on every host having the same file structure
